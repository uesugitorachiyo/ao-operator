# Python SBOM — AO Operator

Generated by `scripts/sbom_python.py`. The companion machine-readable
artifact is `docs/sbom/python-deps.json`.

## Runtime Dependencies

**None required.** The default production path under `scripts/` imports only
the Python standard library. See the header of `requirements-dev.txt` for the
rationale.

## Optional Production Dependencies

Optional production dependencies are lazy and only activated by evidence-pack
features:

| Dependency   | Kind           | Used For                                           | Activation                                      |
|--------------|----------------|----------------------------------------------------|-------------------------------------------------|
| cryptography | python-package | Ed25519 evidence-pack signing and verification     | `--ed25519-private-key` / `--evidence-ed25519-private-key` |
| zstd CLI     | system-binary  | `.tar.zst` evidence-pack archive compression/replay | `--tar-zst` or live-run evidence-pack generation |

## Dev / Test Dependencies

Direct dependencies are declared in `requirements-dev.txt`. Transitive
dependencies are enumerated via `importlib.metadata` against the active
environment, filtered to those whose marker expressions evaluate to True
and that are actually installed.

## License Compatibility

`ao-operator` is dual-licensed under `MIT OR Apache-2.0`. The dependencies
above are compatible with both branches:

- MIT and Apache-2.0 grants compose freely.
- BSD-2-Clause is compatible with both MIT and Apache-2.0.
- The `Apache-2.0 OR BSD-2-Clause` choice on `packaging` lets downstream
  consumers pick either side of our dual license.

## Reproducibility

Run `python3 scripts/sbom_python.py > docs/sbom/python-deps.json` and
diff. Output is deterministic for a fixed environment; the only field
that changes between runs is `generated_at` (ISO 8601 UTC).

## When To Regenerate

- On any change to `requirements-dev.txt`.
- Before tagging a release (added to release readiness in v0.7).
- When upgrading dev tool versions during routine maintenance.
